## 自动化联调

脚本：`qa/tutor-p0-qa.ps1`

前置条件：

- 后端服务已启动，默认地址 `http://127.0.0.1:48080`。
- 数据库已初始化家教 P0 表、城市、字典、菜单权限、积分规则。
- 准备 3 个令牌：管理员、家长 App 用户、教师 App 用户。
- 家长和教师令牌对应的用户最好是 QA 专用账号，脚本会创建/审核发布内容、调整积分、查看联系方式、确认匹配和提交评价。
- 脚本会把令牌原样写入 `Authorization` 请求头；如果你的环境要求 `Bearer xxx`，变量值也要包含 `Bearer ` 前缀。

推荐执行：

```powershell
cd D:\work\jiajiao\zfwl
$env:TUTOR_QA_BASE_URL = "http://127.0.0.1:48080"
$env:TUTOR_QA_TENANT_ID = "1"
$env:TUTOR_QA_ADMIN_TOKEN = "<admin token>"
$env:TUTOR_QA_PARENT_TOKEN = "<parent app token>"
$env:TUTOR_QA_TEACHER_TOKEN = "<teacher app token>"
.\qa\tutor-p0-qa.ps1
```

只检查公开接口和未登录边界：

```powershell
.\qa\tutor-p0-qa.ps1 -SkipMutatingFlow
```

默认行为：

- 脚本会给家长和教师各补 50 积分，确保扣分查看联系方式可执行。
- 脚本会创建并审核一条教师简历、一条家长需求。
- 脚本结束时会下架本次创建的需求和简历，避免重复运行触发“最多 3 条有效发布”限制。
- 如需保留测试数据，增加 `-KeepQaData`。
