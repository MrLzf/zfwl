# 家教积分微信小程序购买设计

## 目标

补齐家教积分的在线购买闭环。家长和教师共用会员积分账户及同一套积分套餐。用户在微信小程序积分页选择后台配置的套餐，完成微信支付后获得基础积分和赠送积分。

首期仅支持微信小程序支付，不开放用户自助退款，不包含微信公众号支付、App 微信支付、支付宝支付和复杂营销活动。

## 安全约束

- 微信商户号、APIv2 密钥、APIv3 密钥、商户私钥和证书不写入源码、SQL 脚本或 Git 历史。
- 微信支付渠道配置通过支付管理后台或部署环境注入。
- 商户私钥和证书放在仓库外目录，配置中仅引用外部路径。
- 本地示例配置只提供占位符。
- 已经通过聊天或其他非密钥管理渠道暴露的密钥需要在微信商户平台轮换后再使用。

## 架构

会员模块继续作为积分账本。积分余额和积分明细仍由 `yudao-module-member` 管理，通过 `MemberPointApi` 写入流水。

家教模块新增积分套餐和积分购买订单，负责套餐管理、创建业务订单、校验支付结果、幂等入账和查询订单状态。

支付能力复用 `yudao-module-pay`：

- 使用 `wx_lite` 微信小程序支付渠道。
- 通过 `PayOrderApi` 或现有支付订单服务创建支付单。
- 复用微信回调验签、支付状态更新和通知任务重试机制。
- 支付模块异步通知家教积分购买业务，不在微信回调线程中直接修改积分。

移动端积分页负责展示套餐、创建订单、提交支付、调用 `uni.requestPayment` 和刷新余额。移动端只提交套餐编号，不提交可信金额或积分数量。

## 数据模型

### 积分套餐

新增表 `tutor_point_recharge_package`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | bigint | 主键 |
| `name` | varchar(64) | 套餐名称 |
| `pay_price` | int | 支付金额，单位分 |
| `point` | int | 基础积分 |
| `bonus_point` | int | 赠送积分，默认为 0 |
| `status` | tinyint | 启用或停用 |
| `sort` | int | 移动端展示顺序 |
| 通用字段 | - | 创建人、创建时间、更新人、更新时间、删除标记、租户编号 |

约束：

- `pay_price` 和 `point` 必须大于 0。
- `bonus_point` 不能小于 0。
- 移动端仅展示启用且未删除的套餐。
- 套餐被历史订单引用后仍可逻辑删除，不影响历史订单展示和支付校验。

### 积分购买订单

新增表 `tutor_point_recharge_order`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | bigint | 主键和业务订单编号 |
| `user_id` | bigint | 会员用户编号 |
| `package_id` | bigint | 下单时选择的套餐编号 |
| `pay_price` | int | 下单时固化的支付金额，单位分 |
| `point` | int | 下单时固化的基础积分 |
| `bonus_point` | int | 下单时固化的赠送积分 |
| `pay_order_id` | bigint | 支付模块订单编号 |
| `pay_status` | bit | 是否支付成功 |
| `pay_channel_code` | varchar(32) | 支付渠道，首期为 `wx_lite` |
| `pay_time` | datetime | 支付完成时间 |
| 通用字段 | - | 创建人、创建时间、更新人、更新时间、删除标记、租户编号 |

约束：

- 下单时复制套餐金额和积分，套餐后续调整不影响历史订单。
- `pay_order_id` 建立唯一索引。
- 订单状态从未支付单向更新为已支付。

## 积分流水

在 `MemberPointBizTypeEnum` 新增积分购买业务类型，例如 `TUTOR_POINT_RECHARGE`。

支付成功后调用：

```java
memberPointApi.addPoint(userId, point + bonusPoint,
        MemberPointBizTypeEnum.TUTOR_POINT_RECHARGE.getType(), orderId.toString());
```

基础积分和赠送积分一次性写入同一条会员积分流水。流水描述需要展示总到账积分，并可包含套餐名称。

## 后端接口

### 移动端接口

新增家教模块 App API：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/app-api/tutor/point-recharge/packages` | 查询启用套餐 |
| `POST` | `/app-api/tutor/point-recharge/orders` | 创建积分购买订单 |
| `POST` | `/app-api/tutor/point-recharge/orders/{id}/pay` | 提交微信小程序支付 |
| `GET` | `/app-api/tutor/point-recharge/orders/{id}` | 查询当前用户的订单状态 |

创建订单入参仅包含 `packageId`。后端从套餐读取金额、基础积分和赠送积分，创建积分购买订单和支付模块订单。

提交支付时，后端限制渠道为 `wx_lite`，获取当前微信小程序用户的 `openid`，并返回拉起支付需要的参数。用户只能提交和查询自己的订单。

### 支付通知接口

新增内部回调接口：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/admin-api/tutor/point-recharge/update-paid` | 支付模块通知积分购买订单已支付 |

接口允许支付模块通知任务访问。服务层必须校验：

- 业务订单存在。
- 支付模块订单存在且状态为支付成功。
- 通知中的支付单编号和业务订单记录一致。
- 支付模块订单的商户订单编号等于积分购买订单编号。
- 支付模块订单金额等于业务订单固化金额。

### 管理端接口

在家教管理下新增积分套餐管理 API：

- 套餐分页查询。
- 创建套餐。
- 编辑套餐。
- 启用或停用套餐。
- 逻辑删除套餐。

管理端金额以元展示，接口和数据库以分保存。

## 支付与入账流程

1. 用户进入移动端积分页，加载启用套餐。
2. 用户选择套餐并确认购买。
3. 家教模块根据套餐编号创建业务订单，固化金额和积分，并创建支付模块订单。
4. 移动端提交业务订单支付，后端使用 `wx_lite` 和当前用户 `openid` 提交支付模块订单。
5. 移动端调用 `uni.requestPayment` 拉起微信支付。
6. 微信回调支付模块，支付模块验签并更新支付单。
7. 支付模块通知任务调用家教模块 `update-paid` 接口。
8. 家教模块校验支付单，通过条件更新将业务订单改为已支付。
9. 家教模块调用 `MemberPointApi.addPoint` 一次性写入基础积分和赠送积分。
10. 移动端支付返回后查询订单状态，到账后刷新用户信息和积分明细。
11. 对微信小程序虚拟物品订单异步上传发货信息。上传失败只记录日志，不回滚积分入账。

## 幂等和事务

积分入账和业务订单状态更新在同一事务中执行。

- 业务订单已经支付，且支付单编号一致：视为重复通知，直接返回成功。
- 业务订单已经支付，但支付单编号不同：拒绝处理并记录错误日志。
- 并发通知通过条件更新 `pay_status = false` 保证只有一个请求成功进入入账流程。
- 支付单状态、编号、商户订单编号或金额不一致：拒绝入账，保留通知任务重试和人工排查线索。
- `MemberPointApi.addPoint` 失败：事务回滚，订单仍为未入账状态，由支付模块通知任务重试。

## 移动端交互

修改 `/pages/user/wallet/score`：

- 增加“购买积分”卡片，展示套餐金额、基础积分、赠送积分和总积分。
- 点击套餐后弹出确认框，再创建订单并拉起微信支付。
- 支付完成后查询订单状态，刷新用户信息和积分明细。
- 用户取消支付时提示“支付未完成”，订单保持未支付，可再次发起。
- 支付失败时提示重试，不在前端本地增加积分。
- 积分不足提示中的主要按钮调整为“购买积分”。

移动端新增积分购买 API 封装，并复用当前微信小程序登录流程保存的 `openid`。

## 管理端交互

在家教管理下新增“积分套餐”页面：

- 套餐列表。
- 新增和编辑套餐。
- 启停套餐。
- 删除套餐。
- 展示金额、基础积分、赠送积分和总积分。

微信商户密钥和证书不在家教套餐页面展示或编辑。它们由支付模块渠道配置管理。

## 支付配置

支付模块新增或配置家教积分购买支付应用标识，并为该应用启用 `wx_lite` 渠道。

部署时配置：

- 微信商户号。
- 微信小程序 AppId。
- 轮换后的 APIv3 密钥。
- 商户证书序列号。
- 仓库外商户私钥路径或安全注入后的私钥内容。
- 公网 HTTPS 支付回调地址。
- 家教积分购买支付应用标识。

APIv2 密钥仅在所选 SDK 或退款能力确实需要时配置。首期不实现退款，不把 APIv2 密钥作为积分购买功能的硬依赖。

## 不包含范围

- 用户自助退款。
- 管理员发起微信退款和退款后扣回积分。
- 微信公众号支付。
- App 微信支付。
- 支付宝支付。
- 用户自定义购买数量。
- 优惠券、限时折扣、会员等级价格和复杂营销活动。

## 测试

### 后端

新增服务单元测试：

- 创建订单时金额和积分来自数据库套餐，客户端不能覆盖。
- 停用或不存在的套餐不能下单。
- 支付成功后增加基础积分和赠送积分之和。
- 重复通知不重复到账。
- 并发条件更新失败时不重复到账。
- 支付单编号、商户订单编号、金额或状态不匹配时拒绝入账。
- 积分写入失败时事务回滚。
- 虚拟物品发货信息上传失败不影响到账。

### 移动端

新增 Node 回归测试：

- 积分页展示购买套餐。
- 下单只提交 `packageId`。
- 支付渠道固定为 `wx_lite`。
- 使用当前微信小程序 `openid` 拉起支付。
- 成功、取消和失败提示正确。
- 支付到账后刷新积分余额和明细。

### 管理端

- 套餐管理 API 封装和页面通过 TypeScript 检查。
- 金额元和分转换正确。
- 启停和删除操作正确刷新列表。

## 验收标准

1. 家长和教师均可在微信小程序积分页看到同一套启用套餐。
2. 用户无法通过修改请求参数篡改金额或积分数量。
3. 微信支付成功后，订单状态变为已支付，会员积分增加基础积分和赠送积分之和。
4. 重复微信回调不会重复增加积分。
5. 支付异常、取消或失败不会增加积分。
6. 支付密钥、商户私钥和证书不会进入 Git 仓库。
7. 管理员可维护积分套餐，套餐调整不影响历史订单。
